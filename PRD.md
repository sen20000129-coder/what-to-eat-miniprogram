## 一、产品概述

### 1.1 产品定位
**今天吃什么**是一款帮助用户解决"每天吃什么"选择困难症的生活服务类微信小程序。核心价值在于通过随机抽取机制，结合用户偏好筛选，提供个性化的美食推荐服务。

### 1.2 目标用户
- 都市上班族：工作繁忙，经常纠结午餐/晚餐吃什么
- 美食爱好者：希望发现新餐厅、新菜式
- 选择困难症患者：需要外力帮助做决定的场景

### 1.3 核心功能矩阵

| 功能模块 | 优先级 | 描述 |
|---------|--------|------|
| 美食随机抽取 | P0 | 核心功能，slot machine 滚动抽取 |
| 美食类型筛选 | P0 | 多选类型标签，精确定位偏好 |
| 时段筛选 | P0 | 早餐/午餐/晚餐/全部 |
| 美食社区 | P1 | 分享美食体验，UGC内容 |
| 个人中心 | P2 | 用户信息、记录统计 |
| 分享卡片 | P2 | 生成美食分享图片 |

### 2.2 【模块1】用餐时段选择

#### 2.2.1 功能描述
允许用户手动选择或自动检测当前用餐时段，用于筛选可抽取的美食范围。

#### 2.2.2 UI规格
- **位置**: 社区入口下方，美食类型选择上方
- **布局**: 横向等分4个tab（全部/早餐/午餐/晚餐）
- **视觉**:
  - 未选中: 米色背景 `#FFF8E7`，深灰文字 `#666`
  - 选中: 金橙色渐变背景 `#FFD700 → #FFA500`，白色文字
  - 圆角: `16rpx`
  - 内边距: `20rpx 16rpx`
  - 阴影（选中态）: `0 6rpx 16rpx rgba(255, 165, 0, 0.3)`

#### 2.2.3 交互规格
| 交互行为 | 触发条件 | 响应结果 |
|---------|---------|---------|
| 点击时段tab | 用户点击 | 切换选中状态，更新timePeriod，重置selectedTypes |

#### 2.2.4 联动规则
```
规则2.2.4.1: 当 timePeriod 变化时
  → 重置 selectedTypes = []
  → 调用 renderTypeTags() 重新渲染类型列表
  → 联动规则 2.3.4.1 自动执行

规则2.2.4.2: timePeriod 取值范围
  → 'all': 全部时段（默认值）
  → 'morning': 早餐 (05:00-10:00)
  → 'lunch': 午餐 (10:00-14:00)
  → 'dinner': 晚餐 (14:00-23:59)
```

#### 2.2.5 数据结构

### 2.4 【模块3】美食抽取滚动区

#### 2.4.1 功能描述
slot machine 风格的食物滚动展示区，用户点击开始抽取，系统自动滚动后暂停。

#### 2.4.2 UI规格
- **位置**: 美食类型选择下方
- **布局**: 垂直滚动，中心高亮选中项
- **视觉**:
  - 可见区域: 5个食物卡片位置
  - 中心项: 放大1.1倍，橙色边框，阴影效果
  - 非中心项: 缩小0.9倍，灰色调
  - 上下遮罩: 渐变透明遮罩
  - 装饰: 顶部/底部小圆点装饰

#### 2.4.3 交互规格
| 交互行为 | 触发条件 | 响应结果 |
|---------|---------|---------|
| 点击开始抽取 | 用户点击 | 启动滚动动画，3秒后自动暂停 |
| 点击暂停 | 滚动中点击 | 立即停止滚动，显示结果 |
| 点击"就这个了" | 结果显示态 | 确认选择，进入冷却 |
| 点击"再来一次" | 结果显示态且retryCount < 3 | 重试抽取 |
| 点击"就这个了" | 第三次重试后 | 显示最终弹窗"这么作还是饿着吧" |

#### 2.4.4 联动规则
```
规则2.4.4.1: 滚动区内容筛选逻辑
  → 调用 getFilteredFoods(selectedTypes, timePeriod)
  → 返回同时满足以下条件的美食列表：
    - 美食.periods 包含 timePeriod 或为 'all'
    - 美食.type 属于 selectedTypes（若 selectedTypes 非空）
  → 使用筛选结果进行抽取

规则2.4.4.2: 初始状态预显示
  → 页面加载时显示5个随机美食
  → 每次筛选条件变化时更新预显示

规则2.4.4.3: 滚动动画数据源
  → 生成20个随机美食用于滚动
  → 从 getFilteredFoods() 结果中随机选取
```

#### 2.4.4 冷却机制
| 场景 | 冷却时长 | 显示内容 |
|-----|---------|---------|
| 点击"就这个了" | 3分钟 | 倒计时卡片，隐藏其他模块 |
| 点击"再来一次"(第3次) | 8分钟 | 倒计时卡片 + 最终弹窗 |

## 三、三层联动核心算法

### 3.1 联动数据流图

```
用户操作 ──▶ 时段选择 ──▶ 重置类型 ──▶ 过滤类型列表
                                    │
                                    ▼
                              渲染类型标签
                                    │
                                    ▼
用户操作 ──▶ 类型选择 ──▶ 筛选美食池 ──▶ 更新抽取区
                                    │
                                    ▼
                              更新筛选状态
```

### 3.2 getFilteredFoods() 算法

```javascript
/**
 * 根据时段和类型筛选美食
 * @param {Array} selectedTypes - 用户选择的类型数组
 * @param {String} timePeriod - 当前选中的时段
 * @returns {Array} 符合条件的美食列表
 */
function getFilteredFoods(selectedTypes, timePeriod) {
  let filtered = allFoods;

  // Step 1: 按时段筛选
  if (timePeriod !== 'all') {
    filtered = filtered.filter(food =>
      food.periods.includes(timePeriod)
    );
  }

  // Step 2: 按类型筛选（多选，支持空选择=全部）
  if (selectedTypes && selectedTypes.length > 0) {
    filtered = filtered.filter(food =>
      selectedTypes.includes(food.type)
    );
  }

  return filtered;
}
```

### 3.3 状态管理规范

```javascript
// 全局状态
state = {
  timePeriod: 'all',      // 当前时段
  selectedTypes: [],      // 已选类型数组
  isRolling: false,       // 是否正在滚动
  retryCount: 0,          // 已重试次数
  maxRetries: 3,          // 最大重试次数
  isCoolingDown: false,   // 是否冷却中
  coolDownSeconds: 0      // 剩余冷却秒数
}
```

### 3.4 联动触发事件清单

| 序号 | 触发事件 | 执行动作 | 影响范围 |
|-----|---------|---------|---------|
| E1 | 切换 timePeriod | selectedTypes = [] | 重置类型选中 |
| E2 | 切换 timePeriod | renderTypeTags() | 过滤类型列表 |
| E3 | 切换 timePeriod | getFilteredFoods() | 更新抽取区预显示 |
| E4 | 切换 timePeriod | updateFilterStatus() | 更新筛选状态文本 |
| E5 | 点击类型标签 | selectedTypes toggle | 切换类型选中态 |
| E6 | 变化 selectedTypes | getFilteredFoods() | 更新抽取区预显示 |
| E7 | 变化 selectedTypes | updateFilterStatus() | 更新筛选状态文本 |
| E8 | 点击开始抽取 | 使用 getFilteredFoods() | 开始抽取动画 |

## 四、美食数据结构

### 4.1 美食数据模型

```javascript
{
  id: Number,           // 唯一ID
  name: String,         // 美食名称（具体，如"重庆老火锅"而非"火锅"）
  type: String,         // 类型名称（需与 foodTypes.name 对应）
  emoji: String,        // Emoji图标
  calories: String,     // 热量描述（如"约800kcal"）
  description: String,  // 简短描述
  tags: Array,          // 标签数组
  comment: String,      // 随机评论
  periods: Array        // 适用时段 ['morning', 'lunch', 'dinner', 'all']
}
```

### 4.2 命名规范
- 美食名称必须具体，禁止使用宽泛名称
- ✅ 正确: "重庆老火锅"、"麻辣香锅"、"铜锅涮肉"
- ❌ 错误: "火锅"、"香锅"、"涮肉"

### 4.3 数据量要求
| 时段 | 最少美食数量 | 示例 |
|-----|------------|------|
| morning | 15种 | 咖啡、面包、包子、煎饼等 |
| lunch | 25种 | 快餐、面点、日料、韩料等 |
| dinner | 30种 | 火锅、烧烤、川菜、韩料等 |
| all (跨时段) | 10种 | 甜品、奶茶、快餐等 |

## 七、错误处理规范

### 7.1 空结果处理

| 场景 | 检测条件 | 处理方式 |
|-----|---------|---------|
| 时段+类型无匹配 | getFilteredFoods().length === 0 | Toast提示"没有找到合适的美食" |
| 美食数据为空 | allFoods.length === 0 | 显示默认提示 |

### 7.2 边界条件

| 条件 | 预期行为 |
|-----|---------|
| selectedTypes = [] | 显示所有类型，抽取全部符合时段的美食 |
| timePeriod = 'all' | 不过滤时段，显示全部美食 |
| 滚动中点击Tab | 禁止操作，需等待滚动结束 |
| 冷却中点击抽取 | 禁止操作，显示倒计时 |
